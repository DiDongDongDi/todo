# 产品文档

## 一句话定位

打开即写、划一下即收进收集箱、每天有一点「完成感」—— 不是功能堆砌的 GTD，而是 **3 秒内能记下一条任务** 的工具。

## 视觉与体验基调

- **界面简单但精致** — 少元素、大留白、排版与动效考究
- **动画要好** — 页面划走、任务落袋、转场均需流畅、有「爽感」
- **打开 App 第一眼就是添加任务** — 默认「收集」Tab，捕获优先

## 设计原则

| 原则 | 含义 |
|------|------|
| 捕获优先 | 默认 Tab 即录入页；所有新任务先进 **收集箱（Inbox）** |
| 多模态录入 | 文字 / 语音转文字 / 贴图 / 录音（后台自动转写） |
| 划走即提交 | 收集 Tab：有内容时 **上划** = 写入收集箱 |
| 大卡片聚焦 | 收集 / 处理 Tab 均以 **单张大卡片** 呈现任务 |
| 手势即命令 | 左右分拣、上下切卡；少按钮 |
| 即时反馈 | 提交、完成、放弃都有动画 + 触觉（haptic） |
| 同步无感 | 离线可用，联网后自动合并；用户不感知「保存」 |
| 克制上瘾 | 用 streak、处理队列清零、轻量成就 —— 不用抽卡/广告式多巴胺 |

## 主导航

底部 Tab 栏，两个 Tab：

| Tab | 名称 | 职责 |
|-----|------|------|
| 1 | **收集** | 空白大卡片录入；有内容时上划保存进收集箱 |
| 2 | **处理** | 收集箱内全部任务以大卡片呈现；分拣、编辑、完成 |

## Tab 1：收集

- 始终只有 **一张大卡片**
- 空白时：可直接输入文字、语音转文字、贴图、录音
- **上划保存**（有内容时）→ 进收集箱 → 卡片归空白 + 提示用户
- 空白时上划无效（回弹 + 轻提示）
- 不在此 Tab 左右滑分拣或浏览历史

## Tab 2：处理

- 收集箱 **全部任务** 自动出现，无需手动移入
- **左滑** → 放弃 → 回收站
- **右滑** → 完成 → 归档
- **上划 / 下划** → 切换下一条 / 上一条
- **可编辑** 任务内容（仅处理 Tab）

## 任务状态

- `inbox` — 收集箱，处理 Tab 展示
- `archived` — 归档（右滑完成）
- `trashed` — 回收站（左滑放弃），可恢复

## 「上瘾」机制

### MVP 包含

1. **处理清零环** — 处理 Tab 待办完成度可视化；全部归档时短庆祝动画
2. **连续打卡 streak** — 每天至少归档 1 条或「处理队列清零」算一天
3. **完成爽感** — 处理 Tab 右滑归档：卡片飞出 + 轻震动；支持撤销（snackbar 3 秒）

### 第二阶段

- 周回顾、轻量徽章（「连续 7 天」「单日完成 20 条」）

## 明确不做（初期）

- 社交排行
- 复杂项目管理（子任务、依赖、甘特图）
- 大量设置项
- 广告式推送轰炸
- 用户自建多列表（初期由 status 虚拟分区驱动）

## 目标平台

Android、iOS、macOS、Windows、Web — Flutter 一套代码。
